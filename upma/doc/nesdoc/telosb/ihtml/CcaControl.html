Interface: CcaControl

interface CcaControl

Controls the radio's CCA and backoff policies.

Author:
Greg Hackmann

Events
event bool getCca(message_t *msg, bool defaultCca) Whether to perform CCA before sending a message must be determined.
event uint16_t getCongestionBackoff(message_t *msg, uint16_t defaultBackoff) The congestion backoff time for a message must be selected.
event uint16_t getInitialBackoff(message_t *msg, uint16_t defaultbackoff) The initial backoff time for a message must be selected.

Events - Details

getCca

event bool getCca(message_t *msg, bool defaultCca)

Whether to perform CCA before sending a message must be determined. If the upper layer has no need to specify this, then return defaultCca.

Parameters:
msg - the message
defaultCca - the radio-default CCA policy
Returns:
whether to perform CCA before sending the message

getCongestionBackoff

event uint16_t getCongestionBackoff(message_t *msg, uint16_t defaultBackoff)

The congestion backoff time for a message must be selected. If the upper layer has no need to change the backoff, then return defaultBackoff.

Parameters:
msg - the message
defaultBackoff - the radio-default backoff length
Returns:
the congestion backoff length, in 32 Khz increments

getInitialBackoff

event uint16_t getInitialBackoff(message_t *msg, uint16_t defaultbackoff)

The initial backoff time for a message must be selected. If the upper layer has no need to change the backoff, then return defaultBackoff.

Parameters:
msg - the message
defaultBackoff - the radio-default backoff length
Returns:
the initial backoff length, in 32 Khz increments
